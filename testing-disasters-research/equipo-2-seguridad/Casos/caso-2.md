# Caso 2 – OWASP Top 10 (Aplicaciones Web – Global)

| **Elemento** | **Descripción** |
|---------------|-----------------|
| **Tipo de falla** | Falta de aplicación de pruebas de seguridad basadas en riesgos conocidos y estándares internacionales. |
| **Causa principal** | En empresas de todo el mundo, se repite el mismo problema: los equipos de desarrollo no siguen las guías del proyecto OWASP Top 10, que identifica las diez vulnerabilidades más comunes en aplicaciones web. Esto ocurre por falta de capacitación, poco tiempo de pruebas o la creencia de que la seguridad “se revisa al final”. |
| **Pruebas faltantes** | Faltaron pruebas estructuradas contra los diez riesgos principales: inyecciones, fallos de control de acceso, exposición de datos sensibles, XSS, configuraciones inseguras, componentes obsoletos, validación de entrada, entre otros. No se realizaron pruebas negativas ni revisiones de código seguro. |
| **Consecuencias** | Muchas empresas lanzan aplicaciones con vulnerabilidades previsibles, lo que termina generando ataques repetitivos, pérdida de datos y sanciones legales por incumplir normativas de protección de información. |
| **Patrón común identificado** | No adoptar un marco de pruebas como OWASP provoca que se repitan los mismos errores en diferentes proyectos. El patrón es claro: no probar según estándares de seguridad conocidos lleva a vulnerabilidades que ya estaban documentadas. |
